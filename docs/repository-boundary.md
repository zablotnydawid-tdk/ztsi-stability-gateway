# Repository Boundary

## Public Repository Purpose

This repository is a public proof-of-work for ZT&SI Cognitive Stability Infrastructure. It demonstrates how an AI Runtime Firewall can evaluate, stabilize, govern, observe, and log candidate model outputs before final manifestation.

Public code should remain understandable, dependency-light, testable, and suitable for review by external engineers.

## Private Repository Purpose

Private repositories are reserved for advanced sovereign-core research, proprietary projection mechanics, strategic governance methods, production hardening, private experiments, and deployment-specific enforcement logic.

## What Belongs In Public

- Gateway architecture and runtime flow.
- Lightweight semantic drift and coherence scoring.
- Public projection demonstrations.
- Configurable policy examples.
- Multi-agent governance mesh demonstrators.
- Local JSON and JSONL persistence.
- Tests, docs, public examples, and OpenAPI examples.

## What Belongs In Private

- Advanced sovereign-core IP.
- Proprietary adaptive projection mechanics.
- Strategic governance algorithms.
- Sensitive deployment configurations.
- Provider credentials, API keys, and operational secrets.
- Private datasets or private model behavior traces.

## IP Protection Doctrine

The public repository should show the shape, discipline, and engineering credibility of the ZT&SI runtime without revealing private mechanisms that create strategic advantage. Public work demonstrates capability; private work preserves protected depth.
