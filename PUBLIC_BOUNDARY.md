# Public Boundary

## Public Repository Contains

- Demonstrator runtime
- FastAPI API layer
- Mock LLM adapter
- Semantic drift intelligence
- Bounded projection
- Policy governance
- Memory and lineage demo
- Runtime telemetry
- Multi-agent governance mesh
- Local JSON and JSONL persistence
- Tests, docs, and public examples

## Private Sovereign Core Contains

- Advanced adaptive projection
- Distributed runtime consensus
- Proprietary governance calibration
- Semantic hypervisor research
- Internal strategic doctrine
- Production hardening roadmap

## Boundary Rule

The public repository should demonstrate the architecture, engineering discipline, and runtime governance behavior of ZT&SI Cognitive Stability Infrastructure. It should not expose private sovereign-core research, advanced adaptive projection mechanics, proprietary arbitration methods, private credentials, or unreleased internal doctrine.

Public code is allowed to be understandable and testable. Private systems are where strategic depth, production enforcement, and proprietary methods belong.
