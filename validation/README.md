# Governance Validation Layer

The Governance Validation Layer turns the public ZT&SI Stability Gateway into a controlled runtime governance laboratory.

It does not add new intelligence, AGI-like behavior, hidden projection mechanics, or private sovereign-core methods. It validates how the existing runtime behaves under deterministic degradation, recursion pressure, policy stress, telemetry gaps, lineage reconstruction, and repository boundary constraints.

Core principle:

```text
The system is not stable because it runs.
The system is stable because its behavior during degradation is known.
```

## Areas

- `runtime/`: deterministic runtime scenarios, recursive failure simulations, drift escalation chains, agent conflict simulations, and rollback integrity checks.
- `replay/`: historical runtime event replay, policy outcome comparison, rollback consistency checks, and lineage reconstruction verification.
- `policies/`: contradictory policy tests, recursion saturation tests, drift budget exhaustion, malformed agent output simulation, and governance deadlock scenarios.
- `telemetry/`: telemetry consistency checks, missing event detection, orphan lineage detection, and runtime state desync detection.
- `boundary/`: public/private split validation, sovereign leakage scan, runtime secret exposure scan, and repository boundary audit.

## Public Boundary

This directory belongs in the public repository because it validates public runtime behavior. It must not include private sovereign-core research, proprietary calibration, advanced adaptive projection, internal doctrine, credentials, runtime dumps, or hidden production hardening methods.
